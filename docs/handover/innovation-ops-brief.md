## What is a processogram

A processogram is a scientific diagram produced by the Welfare Footprint Institute (WFI)
that maps the living conditions of farmed animals across a production system — from birth
or arrival to slaughter or end of productive life.

Each processogram is organized in four levels, from broadest to most specific:

```
Production System    — the overall farming context (e.g. indoor pig farming)
└── Life-Fate        — a major life stage or fate (e.g. sow in farrowing crate)
    └── Phase        — a specific period within that life stage (e.g. lactation)
        └── Component / Indicator   — a concrete element the user can click and explore
                                      (e.g. a specific animal, cage, conveyor, or condition)
```

The Component/Indicator level (the deepest level) is where users interact with the
diagram — clicking an element opens a description, scientific context, and related
questions generated by AI.

Processograms are drawn by Jean (SVG designer) and exported as SVG files. The files
are uploaded to the platform by Wladimir or Gabriel through an admin panel.

---

## What is the WelfareData platform

WelfareData (welfaredata.org) is the WFI's platform for navigating processograms
online. It allows researchers, policymakers, and the public to explore the conditions
animals experience at each stage of production.

The platform has two main parts:

- **The viewer** — the public-facing page where users navigate processograms
- **The admin panel** — the internal interface where Wladimir and Gabriel upload SVGs,
  trigger AI analysis, and manage content

Both parts are built and maintained by Gabriel as part of the WelfareData-New project.

## Project history

### 1. The Herikle system (legacy — being replaced)

The original processogram viewer was built by Herikle, a developer who worked on an
earlier version of the platform. It runs on Vercel (a cloud hosting service) under the
domain `app.welfaredata.org`.

The system works, but it has a significant performance problem: on complex processograms
with hundreds of elements, each click takes approximately 800 milliseconds to respond.
For a research tool where users frequently navigate through many levels, this is
noticeably slow and degrades the experience.

This system is being replaced. Herikle is aware and has one pending action when the
time comes: disconnect the domain `app.welfaredata.org` from his personal Vercel account
so it can be pointed to the new system.

### 2. WelfareData-New (the full platform — ready, awaiting the navigator)

In parallel with the Herikle system, Gabriel built WelfareData-New — a complete rebuild
of the platform from scratch. It includes:

- A backend that handles SVG uploads, AI analysis (via Google Gemini), user
  authentication, and data storage
- A frontend with the viewer, admin panel, and AI chat interface
- Storage on Google Cloud (SVG files, processed images)
- A database (MongoDB) storing all content and metadata

WelfareData-New is feature-complete and runs correctly in a local development
environment. It is not yet in production. The only missing piece is the navigation
engine — the component that lets users click through the processogram hierarchy
interactively. That is what the Canvas Navigator provides.

### 3. The Canvas Navigator (the new engine — integrated)

The Canvas Navigator is the new interactive navigation engine, developed by Gabriel.
It replaces the slow navigation approach of the Herikle system with a fundamentally
different technical approach that eliminates the performance problem entirely.

From the user's perspective, the experience is the same: click an element, zoom in,
navigate deeper, use the breadcrumb to go back. What changes is what happens under
the hood — the new engine is fast, smooth at 60 frames per second, and scales to
processograms of any complexity without degradation.

The Canvas Navigator has been validated with Cattle v20 and Hatchery v30, and integrated
into WelfareData-New. Full navigation is working in the platform. The remaining SVGs
(Laying Hens, Pig, Broilers) are pending Jean's structural corrections and can be
validated in parallel with the ongoing stabilization.

### The preflight tool

To prevent these problems from recurring, Gabriel built a preflight checker
(`tools/svg-preflight.html`) — a standalone tool that Jean opens in a browser,
drops an SVG file onto, and immediately receives a report of any issues.

From June 2026 onward, every SVG must pass the preflight before being delivered.
Issues marked as errors must be fixed. Issues marked as warnings must be explained.

### What is accepted as Phase 1 complete

Phase 1 is considered complete when Cattle and Hatchery are accepted (already done)
and the integration spike into WelfareData-New is underway. Laying Hens, Pig, and
Broilers can be validated in parallel — they do not block the integration.

## Key principle to keep in mind

The Canvas Navigator separates three concerns that must never be mixed:

- **Visual art** — what Jean draws. Jean's domain.
- **Semantic structure** — what each element means scientifically. WFI's domain.
- **Interaction** — how the engine responds to user input. Gabriel's domain.

Problems arise when these layers are conflated — for example, when a visual design
decision is treated as a technical fix, or when a technical limitation is solved by
changing what something means scientifically. Part of the Operations Lead role is
recognizing when a conversation is crossing these lines and routing it to the right
person.
